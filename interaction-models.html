Overview

Interaction models help designers predict user behavior and evaluate interface efficiency. They provide structured frameworks for analyzing tasks and system feedback.

Norman’s Model of Interaction

Norman’s model describes interaction as a cycle of goal formation, execution, and evaluation. The gulfs of execution and evaluation highlight gaps between user intentions and system responses (Dix et al., Chapter 7).

Example: Clicking "Submit" and receiving confirmation demonstrates execution and evaluation steps.

GOMS Model

GOMS (Goals, Operators, Methods, Selection rules) predicts expert user performance by breaking tasks into measurable components (Dix et al., Chapter 8).

Example: Filling a form using a consistent sequence of keystrokes shows GOMS in practice.

Keystroke-Level Model (KLM)

KLM estimates task completion time based on low-level actions like keystrokes, pointing, and mental preparation (Dix et al., Chapter 8).

Example: Calculating the time to copy text between fields using KLM operators.

Reflection

Interaction models guide usability evaluation and interface design by identifying inefficiencies and reducing cognitive load.
